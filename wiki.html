01 · prompt injection

prompt injection.

An attack class where adversarial input subverts an LLM's intended instructions to execute attacker goals — same logic-injection pattern as SQLi or XSS, different substrate.

1.1

Direct Injection

Adversarial instructions placed directly into the user input channel. The simplest variant: "Ignore previous instructions and...". Modern systems filter naive forms but remain vulnerable to encoded, role-played, or multi-instruction payloads.

payload
Single-turn instruction in the user channel.

1.2

Indirect Injection

Instructions hidden in content the model later retrieves — emails, web pages, RAG documents, tool outputs. The user is not the attacker; the data is. Most agentic systems are vulnerable by design.

vector
Email body, scraped page, vector-DB doc, file metadata, image OCR.

defense
Strict trust-boundary marking, output-channel separation, instruction quarantine.
1.3

Multi-Turn Injection

Adversarial instructions decomposed across multiple turns to bypass single-turn classifiers. State accumulates; each turn appears benign. The exploit lives in the conversation graph, not any single message.

02 · jailbreaking

jailbreaking.

Bypassing an LLM's alignment layer to produce content the model is trained to refuse. Distinct from prompt injection: jailbreaking targets the safety policy; injection targets the instruction hierarchy.

2.1

Persona Attacks

Inducing the model to adopt a fictional identity for which the safety policy purportedly does not apply. DAN, "evil assistant", role-playing system prompts. Largely patched in frontier models but still effective in chained or compositional form.

aatmf tactic
T3 — Instruction Hijack · read more
2.2

Encoding & Obfuscation

Base64, ROT13, leetspeak, Unicode confusables, language-switch, ASCII art. The classifier sees harmless tokens; the model decodes and complies. Effectiveness drops as decoding-aware classifiers improve, but new encodings emerge faster than they're patched.

aatmf tactic
T7 — Output Manipulation & Exfiltration · read more
2.3

Compositional Bypass — P.R.O.M.P.T

Treating jailbreaks as a grammar — combining persona, encoding, instruction-hijack, and turn-decomposition primitives into compositional attacks. The whole framework is documented at P.R.O.M.P.T.

2.4

Decay & Drift

Long conversations drift the model away from its system prompt. Safety fidelity decays with context length. Quantified in The LLM Red Teamer's Playbook.

05 · tool & agent abuse

tool & agent abuse.

06 · mcp vs a2a

mcp vs a2a.

Two protocols, two trust models. MCP exposes tools to a single agent; A2A delegates between agents. The trust boundary lives in different places — and so do the attacks.

mcp
Model Context Protocol — standardized tool/data exposure to LLMs.
a2a
Agent-to-Agent — autonomous agents delegating to each other.
trust delta
MCP trusts the user; A2A trusts another agent's identity. Both can be spoofed.

07 · defense layers

defense layers.

The Red Teamer's Playbook treats defense as a five-layer chain. Bypassing one is rarely sufficient; bypassing all is necessary for full exploitation.

layer 1 — input filters
Pattern matchers, classifiers, embedding-based detectors. First line, weakest layer.
layer 2 — alignment
The model's own learned safety policy. Strong on common cases, brittle on the long tail.
layer 3 — identity
System prompt and persona enforcement. Decays with context length.
layer 4 — output
Post-generation classifiers and filters. The 30% blind spot lives here. · read more
layer 5 — agentic trust
Tool-call authorization, memory write controls, inter-agent trust.
